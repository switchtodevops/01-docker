The docker build command builds Docker images from a Dockerfile and a "context". The context is the set of files located in the specified PATH or URL.

Basic Syntax

docker build [OPTIONS] PATH | URL | -

Common Commands

Build image with tag

docker build -t myapp:latest .

Builds an image named "myapp" with tag "latest" from the current directory

Build with custom Dockerfile

docker build -f Dockerfile.prod -t myapp:prod .

Uses a specific Dockerfile for production build

Build with build arguments

docker build --build-arg VERSION=1.0 -t myapp:1.0 .

Passes build-time variables to the Dockerfile

Important Options

  • -t, --tag: Name and optionally tag the image
  • -f, --file: Name of the Dockerfile
  • --no-cache: Build without using cache
  • --build-arg: Set build-time variables
  • --target: Set target build stage (multi-stage builds)
Pro Tip: Use .dockerignore file to exclude files and directories from the build context to speed up builds and reduce image size.

Multi-Stage Build Example

# Build stage FROM node:18 AS builder COPY . /app RUN npm install && npm run build # Production stage FROM node:18-alpine COPY --from=builder /app/dist /app CMD ["node", "/app/server.js"]
Best Practice: Always tag your images with specific versions instead of using "latest" in production.

The docker push command uploads a Docker image to a registry (Docker Hub, private registry, etc.) so it can be shared and deployed.

Basic Syntax

docker push [OPTIONS] NAME[:TAG]

Common Commands

Push to Docker Hub

# Login first docker login # Push image docker push username/myapp:1.0.0

Pushes image to Docker Hub under your username

Push multiple tags

docker push username/myapp:1.0.0 docker push username/myapp:latest

Pushes same image with different tags

Push to private registry

# Login to private registry docker login registry.company.com # Push image docker push registry.company.com/myapp:1.0.0

Pushes to company's private Docker registry

Complete Workflow

# Step 1: Build the image docker build -t myapp:1.0.0 . # Step 2: Tag for Docker Hub docker tag myapp:1.0.0 username/myapp:1.0.0 docker tag myapp:1.0.0 username/myapp:latest # Step 3: Login to Docker Hub docker login # Step 4: Push images docker push username/myapp:1.0.0 docker push username/myapp:latest

Important Notes

  • Login Required: Must login before pushing
  • Proper Tagging: Image must be tagged with registry/username
  • Network Required: Needs internet connection
  • Permissions: Must have push access to repository
  • Image Layers: Only changed layers are uploaded
Pro Tip: Docker only pushes layers that don't already exist in the registry, making subsequent pushes much faster.

CI/CD Integration

# GitHub Actions example - name: Login to Docker Hub uses: docker/login-action@v3 with: username: ${{ secrets.DOCKER_USERNAME }} password: ${{ secrets.DOCKER_PASSWORD }} - name: Build and push uses: docker/build-push-action@v5 with: context: . push: true tags: | username/myapp:${{ github.sha }} username/myapp:latest
Best Practice: In CI/CD pipelines, use service accounts or access tokens instead of personal credentials for pushing images.

Verify Push

# Pull the image to verify docker pull username/myapp:1.0.0 # Run the pulled image docker run -d -p 8080:80 username/myapp:1.0.0

Registry Options

  • Docker Hub: docker.io/username/image
  • AWS ECR: aws_account_id.dkr.ecr.region.amazonaws.com/image
  • Google GCR: gcr.io/project-id/image
  • Azure ACR: registry.azurecr.io/image
  • GitHub: ghcr.io/username/image

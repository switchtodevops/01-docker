The docker run command creates and starts a new container from a Docker image. It's one of the most commonly used Docker commands.

Basic Syntax

docker run [OPTIONS] IMAGE [COMMAND] [ARG...]

Common Commands

Run container in detached mode

docker run -d --name myapp myapp:latest

Runs container in background with a custom name

Run with port mapping

docker run -d -p 8080:80 nginx

Maps host port 8080 to container port 80

Run with environment variables

docker run -d -e DB_HOST=localhost -e DB_PORT=5432 myapp

Passes environment variables to the container

Run with volume mount

docker run -d -v /host/data:/container/data myapp

Mounts host directory into container for persistent data

Important Options

  • -d, --detach: Run container in background
  • -p, --publish: Publish container port to host
  • -e, --env: Set environment variables
  • -v, --volume: Mount volumes
  • --name: Assign a name to container
  • --rm: Remove container when it exits
  • -it: Interactive terminal (for debugging)
  • --restart: Restart policy (no, on-failure, always, unless-stopped)
Pro Tip: Use `docker run --rm` for temporary containers that you want to be automatically removed after they stop.

Production Example

docker run -d \ --name myapp-prod \ -p 80:8080 \ -e NODE_ENV=production \ -e DB_HOST=db.example.com \ --restart unless-stopped \ --health-cmd "curl -f http://localhost:8080/health || exit 1" \ --health-interval=30s \ myapp:1.0.0
Best Practice: Always specify exact image versions (e.g., myapp:1.0.0) instead of "latest" in production environments.

Quick Reference

# Run and enter container shell docker run -it ubuntu bash # Run with automatic cleanup docker run --rm alpine echo "Hello World" # Run with resource limits docker run -d --memory="512m" --cpus="1.5" myapp
